00025 /*
00026  * Compressed packet format:
00027  *
00028  * The first octet contains the packet type (top 3 bits), TCP
00029  * 'push' bit, and flags that indicate which of the 4 TCP sequence
00030  * numbers have changed (bottom 5 bits).  The next octet is a
00031  * conversation number that associates a saved IP/TCP header with
00032  * the compressed packet.  The next two octets are the TCP checksum
00033  * from the original datagram.  The next 0 to 15 octets are
00034  * sequence number changes, one change per bit set in the header
00035  * (there may be no changes and there are two special cases where
00036  * the receiver implicitly knows what changed -- see below).
00037  *
00038  * There are 5 numbers which can change (they are always inserted
00039  * in the following order): TCP urgent pointer, window,
00040  * acknowlegement, sequence number and IP ID.  (The urgent pointer
00041  * is different from the others in that its value is sent, not the
00042  * change in value.)  Since typical use of SLIP links is biased
00043  * toward small packets (see comments on MTU/MSS below), changes
00044  * use a variable length coding with one octet for numbers in the
00045  * range 1 - 255 and 3 octets (0, MSB, LSB) for numbers in the
00046  * range 256 - 65535 or 0.  (If the change in sequence number or
00047  * ack is more than 65535, an uncompressed packet is sent.)
00048  */
00049 
00050 /*
00051  * Packet types (must not conflict with IP protocol version)
00052  *
00053  * The top nibble of the first octet is the packet type.  There are
00054  * three possible types: IP (not proto TCP or tcp with one of the
00055  * control flags set); uncompressed TCP (a normal IP/TCP packet but
00056  * with the 8-bit protocol field replaced by an 8-bit connection id --
00057  * this type of packet syncs the sender & receiver); and compressed
00058  * TCP (described above).
00059  *
00060  * LSB of 4-bit field is TCP "PUSH" bit (a worthless anachronism) and
00061  * is logically part of the 4-bit "changes" field that follows.  Top
00062  * three bits are actual packet type.  For backward compatibility
00063  * and in the interest of conserving bits, numbers are chosen so the
00064  * IP protocol version number (4) which normally appears in this nibble
00065  * means "IP packet".
00066  */
00067
