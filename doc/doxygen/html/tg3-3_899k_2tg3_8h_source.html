02080 /* There are two ways to manage the TX descriptors on the tigon3.
02081  * Either the descriptors are in host DMA'able memory, or they
02082  * exist only in the cards on-chip SRAM.  All 16 send bds are under
02083  * the same mode, they may not be configured individually.
02084  *
02085  * This driver always uses host memory TX descriptors.
02086  *
02087  * To use host memory TX descriptors:
02088  *  1) Set GRC_MODE_HOST_SENDBDS in GRC_MODE register.
02089  *     Make sure GRC_MODE_4X_NIC_SEND_RINGS is clear.
02090  *  2) Allocate DMA'able memory.
02091  *  3) In NIC_SRAM_SEND_RCB (of desired index) of on-chip SRAM:
02092  *     a) Set TG3_BDINFO_HOST_ADDR to DMA address of memory
02093  *        obtained in step 2
02094  *     b) Set TG3_BDINFO_NIC_ADDR to NIC_SRAM_TX_BUFFER_DESC.
02095  *     c) Set len field of TG3_BDINFO_MAXLEN_FLAGS to number
02096  *            of TX descriptors.  Leave flags field clear.
02097  *  4) Access TX descriptors via host memory.  The chip
02098  *     will refetch into local SRAM as needed when producer
02099  *     index mailboxes are updated.
02100  *
02101  * To use on-chip TX descriptors:
02102  *  1) Set GRC_MODE_4X_NIC_SEND_RINGS in GRC_MODE register.
02103  *     Make sure GRC_MODE_HOST_SENDBDS is clear.
02104  *  2) In NIC_SRAM_SEND_RCB (of desired index) of on-chip SRAM:
02105  *     a) Set TG3_BDINFO_HOST_ADDR to zero.
02106  *     b) Set TG3_BDINFO_NIC_ADDR to NIC_SRAM_TX_BUFFER_DESC
02107  *     c) TG3_BDINFO_MAXLEN_FLAGS is don't care.
02108  *  3) Access TX descriptors directly in on-chip SRAM
02109  *     using normal {read,write}l().  (and not using
02110  *         pointer dereferencing of ioremap()'d memory like
02111  *     the broken Broadcom driver does)
02112  *
02113  * Note that BDINFO_FLAGS_DISABLED should be set in the flags field of
02114  * TG3_BDINFO_MAXLEN_FLAGS of all unused SEND_RCB indices.
02115  */
